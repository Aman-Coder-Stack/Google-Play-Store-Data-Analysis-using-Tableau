# Google-Play-Store-Data-Analysis-using-Tableau

📌 Project Overview

This project focuses on analyzing Google Play Store applications using Tableau to uncover meaningful insights about app performance, user engagement, and market trends. The dashboard provides an interactive and visual way to explore data related to installs, ratings, reviews, categories, and user sentiment.

🎯 Objectives

Analyze overall app performance across different categories
Compare Free vs Paid applications
Understand user behavior using ratings and reviews
Identify top-performing categories based on installs and app count
Explore sentiment distribution of user reviews
📊 Dashboard Features

KPI metrics:

Total Apps
Total Installs
Total Reviews
Average Rating
Paid Apps Percentage
Interactive Filters:

Category
Type (Free/Paid)
Sentiment
Update Year
Visualizations:

Bar Charts (Category comparison, installs)
Scatter Plots (Installs vs Rating, Reviews vs Rating)
Histogram (Rating distribution)
🔍 Key Insights

A large majority of apps on the Play Store are free, while paid apps represent a very small percentage.
High installs do not always correspond to high ratings, indicating that popularity does not always reflect user satisfaction.
Certain categories dominate both in terms of number of apps and total installs.
Most user sentiments are positive, showing general user satisfaction with apps.
🛠 Tools & Technologies Used

Tableau (for dashboard creation and visualization)
Microsoft Excel / CSV (for dataset handling and preprocessing)
